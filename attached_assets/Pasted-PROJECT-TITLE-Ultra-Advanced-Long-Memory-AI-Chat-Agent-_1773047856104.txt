PROJECT TITLE: Ultra Advanced Long-Memory AI Chat Agent (ChatGPT-Level but Stronger)

OBJECTIVE:
Build a production-grade AI Chat Web Application similar to ChatGPT and Gemini, but with EXTREMELY LONG PER-CHAT MEMORY, HIGH SPEED, HONEST RESPONSES, and EXPERT-LEVEL WEB DEVELOPMENT INTELLIGENCE.

This must be a complete, deployable, full-stack system.

-----------------------------------------
CORE REQUIREMENTS
-----------------------------------------

1) CHAT BEHAVIOR
- Agent must behave like ChatGPT in conversation.
- Must read FULL CHAT HISTORY before answering.
- Must not only rely on last few messages.
- Must analyze entire chat context before generating reply.
- Must NEVER bluff or hallucinate.
- If unsure → clearly say “I don’t know”.
- Must respond in the same language as the user automatically.
- Must support ALL major languages (English, Urdu, Hindi, Arabic, Vietnamese, etc).

2) EXTREME LONG MEMORY (VERY IMPORTANT)
- Each chat must support extremely long memory.
- If technically possible, allow very large pasted input (millions of lines).
- If input is too large for one request:
  - Implement chunking.
  - Store content in parts.
  - Reconstruct intelligently.
- Use:
  - Full message history
  - Rolling summary memory
  - Optional semantic retrieval (embeddings)
- The AI must answer after reviewing the entire conversation.
- Memory must be per-chat and isolated.

3) PERFORMANCE
- Streaming responses (real-time typing effect).
- Extremely fast response pipeline.
- Optimized rendering (no full page re-render on every message).
- Virtualized message list for huge chats.
- Efficient database queries with indexing.

4) UI REQUIREMENTS
- Modern ChatGPT-style interface.
- Sidebar with chat list.
- Create new chat button.
- Delete chat option.
- Dark Mode toggle button.
- Light/Dark preference saved in localStorage.
- Fully responsive (mobile + desktop).
- Smooth scrolling and polished UX.

5) INTELLIGENCE LEVEL
The AI must act like:
- A top-class senior web developer.
- Detect coding mistakes instantly from a single line.
- Explain the bug clearly.
- Provide clean and correct solution.
- Detect:
  - TypeScript errors
  - Logic bugs
  - Prisma schema issues
  - API mistakes
  - Performance problems

6) TRUTHFULNESS POLICY
- Never fabricate APIs.
- Never invent documentation.
- Never fake knowledge.
- If data is missing → clearly state uncertainty.
- Honesty is mandatory.

-----------------------------------------
DATABASE STRUCTURE
-----------------------------------------

Use STRING IDs everywhere (NO integers).

Models:

User
- id (String, cuid)
- email (unique)
- name (optional)
- chats (relation)
- createdAt

Chat
- id (String, cuid)
- title (optional)
- userId (required)
- relation to User
- messages (relation)
- createdAt

Message
- id (String, cuid)
- chatId (required)
- relation to Chat
- role ("user" | "assistant" | "system")
- content (TEXT)
- createdAt

Optional Advanced:
ChatSummary
- id
- chatId (unique)
- summary (TEXT)

IMPORTANT RELATION RULE:
If foreign key is required → relation must be required.
If foreign key is optional → relation must be optional.
Never mismatch optionality.

-----------------------------------------
API REQUIREMENTS
-----------------------------------------

POST /api/chat        → create chat
GET  /api/chat/:id    → get full chat with messages
POST /api/message     → add message + generate AI reply (streamed)
GET  /api/chats       → list user chats

Must be fully type-safe.
